# US-001: Project Scaffolding

**Status:** tested
**Model:** sonnet
**Component:** System
**Depends On:** None
**Traces To:** NFR-001, NFR-002, NFR-003, NFR-004, NFR-005, NFR-008

## Description

Establishes the complete Python project structure for SEFI@Home, including pyproject.toml with all dependencies and Python version constraints, the `src/sefi/` package hierarchy with empty `__init__.py` files for all subpackages, a top-level `tests/` tree, and a `data/.gitkeep` placeholder. This story delivers the skeleton that all other stories build upon; nothing functional is implemented here.

## Files to Create / Modify

- `pyproject.toml` — project metadata, `python_requires = ">=3.10"`, all MVP dependencies (fastapi, uvicorn, pydantic, pytest, mypy, pydocstyle)
- `src/sefi/__init__.py` — package root with module docstring and version constant
- `src/sefi/config.py` — `AppConfig` dataclass/Pydantic model reading from environment variables (data dir, DB path, log level)
- `src/sefi/db/__init__.py` — subpackage init with docstring
- `src/sefi/generator/__init__.py` — subpackage init with docstring
- `src/sefi/api/__init__.py` — subpackage init with docstring
- `src/sefi/validation/__init__.py` — subpackage init with docstring
- `src/sefi/store/__init__.py` — subpackage init with docstring
- `tests/__init__.py` — test root init
- `tests/unit/__init__.py` — unit test subpackage init
- `tests/integration/__init__.py` — integration test subpackage init
- `data/.gitkeep` — keeps the data/ directory tracked in git without committing data files

## Acceptance Criteria

- [ ] `python --version` in the project virtualenv returns 3.10 or higher (NFR-001)
- [ ] `pyproject.toml` contains `python_requires = ">=3.10"` (NFR-001)
- [ ] `mypy src/` in strict mode reports zero "Missing type annotation" errors against the scaffolding files (NFR-002)
- [ ] All `__init__.py` and `config.py` files include module-level docstrings; `pydocstyle src/` reports zero missing-docstring errors (NFR-008)
- [ ] `pytest tests/` runs without configuration errors (NFR-005)
- [ ] All API data-model stubs (if any in config.py) are defined as `pydantic.BaseModel` subclasses (NFR-003)
- [ ] No ORM imports exist anywhere in the scaffolding; sqlite3 is the only DB library imported (NFR-004)

## Notes

- `config.py` should expose `get_config() -> AppConfig` that reads `DATA_DIR`, `FINDINGS_DB_PATH`, and `LOG_LEVEL` from environment, with sensible defaults (`data/`, `data/findings.db`, `INFO`).
- Keep `__init__.py` files minimal — a one-line docstring and optional `__all__ = []` is sufficient.
- Do not add any runtime logic to `__init__.py` files; side effects on import cause test isolation problems.
- `pyproject.toml` should define `[tool.pytest.ini_options]` pointing to `tests/` and `[tool.mypy]` with `strict = true`.
